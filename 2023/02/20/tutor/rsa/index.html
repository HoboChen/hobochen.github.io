This is some code samples/briefing of my RSA slides.

concepts

public key: \((n, e)\); \(e\) is typically 65537 private key: \((n, d)\)

\(p, q\) is large primes.

truth:

\[n = pq\] \[de \equiv 1 \text{ mod } \varphi(n) \]

It is hard to get \(\varphi(n)\) from \(n\) without knowing \(p\) and \(q\).

extended wiki page

for length diff of p and q:

https://crypto.stackexchange.com/questions/35087/should-rsa-primes-p-and-q-differ-in-length-by-a-few-digits

for why e has less hamming weight would benefit effiency:

https://en.wikipedia.org/wiki/Hamming_weight

In modular exponentiation by squaring, the number of modular multiplications required for an exponent e is log2 e + weight(e). This is the reason that the public key value e used in RSA is typically chosen to be a number of low Hamming weight.

sign and verify

concept

sign:

message is m, use private key to encrypt h(m), got s;

verify:

use public key to decrypt s; see if equals h(m)

fun fact:

you can use private key to encrpt or decrypt, depends on which operation you want to protect; as encrypt and decrypt are dual operations.

if you choose to use a long enough public key (rather than 65537, aka, keeping public key as a secret as well), then the 2 sides can both encrypt/decrypt -- just like symmetric encryption.
